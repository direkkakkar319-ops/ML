Forward vs Reverse Mode Autodiff

Step through both modes on the same example function.

Forward mode: sweep left→right, carrying ∂v/∂x₁ (tangent) forward at each step.
Example:  f(x₁, x₂) = ln(x₁) + x₁·x₂ − sin(x₂)  at  x₁=2, x₂=5  → find ∂f/∂x₁
Forward mode — when to use
Few inputs, many outputs (e.g. sensitivity analysis, Jacobian columns). Cost scales with number of inputs.
Reverse mode — when to use
Many inputs, few outputs (e.g. neural net with millions of params, one loss). Cost scales with number of outputs. This is backpropagation.
Property Forward Reverse
Cost per gradient O(inputs) passes O(outputs) passes
Memory Low (no tape) Higher (caches forward values)
ML training Impractical at scale The standard (backprop)
Sensitivity analysis Great fit Overkill
PyTorch equivalent torch.autograd.functional.jvp() loss.backward()
